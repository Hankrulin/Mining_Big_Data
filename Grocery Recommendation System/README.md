# Grocery Recommendation System

This project develops a recommendation system for grocery transaction data using a combination of pattern mining, collaborative filtering, and hybrid recommendation techniques. The goal is to identify purchasing patterns and generate product recommendations based on customers’ historical transactions.

The project is structured into three main tasks:

1. Frequent pattern discovery and basket analysis  
2. Collaborative filtering recommendation  
3. Hybrid recommendation system combining multiple recommendation signals

## Project Overview

Retail transaction data often contains useful patterns about which products are frequently purchased together and which products are likely to interest a customer based on purchasing behaviour. In this project, multiple recommendation approaches are implemented and compared using grocery transaction data.

The project begins with transaction preprocessing and exploratory analysis, then builds several recommendation strategies, including frequent itemset-based recommendation, user-based collaborative filtering, SVD-based recommendation, co-occurrence recommendation, and a final hybrid recommendation model.

## Project Structure

### Task 1: Pattern Discovery

This task focuses on transaction pattern analysis and association mining.

Main steps include:

- data reading and preprocessing
- transaction grouping
- exploratory data analysis
- top purchased items analysis
- Apriori frequent itemset mining
- support analysis for length-2 itemsets
- item co-occurrence visualisation
- top members’ frequent pair analysis

The aim of this task is to understand which products are frequently purchased together and to generate rule-based recommendation signals.

### Task 2: Collaborative Filtering

This task develops recommendation models based on user-item interactions.

The following recommendation approaches are implemented:

- **User-Based Collaborative Filtering**
  - cosine similarity is used to identify similar users
  - recommendations are generated from neighbours’ purchasing behaviour

- **Model-Based Collaborative Filtering using Truncated SVD**
  - latent factor decomposition is applied to the user-item matrix
  - recommendations are generated from reconstructed user-item preferences

- **Pattern-Based Recommendation**
  - recommendations are generated using frequent itemsets from Task 1

- **Co-Occurrence-Based Recommendation**
  - recommendations are generated from products that commonly appear together in transactions

Each method is evaluated using coverage, average recommendation length, and novelty. :contentReference[oaicite:2]{index=2}

### Task 3: Hybrid Recommendation System

This task combines multiple recommendation sources into a final hybrid recommender.

The hybrid system integrates signals from:

- collaborative filtering model 1
- collaborative filtering model 2
- frequent pattern relationships
- item co-occurrence information

The enhanced hybrid design also includes:

- temporal user profiling
- weighted scoring of recommendation sources
- fallback strategies for general recommendation coverage

## Methods Used

### 1. Apriori Frequent Pattern Mining
Apriori is used to discover frequent itemsets and identify products that often appear together in the same basket. This provides interpretable recommendation signals and supports pattern-based recommendation.

### 2. User-Based Collaborative Filtering
Cosine similarity is used to compare users based on their purchase histories. Recommendations are generated by finding users with similar behaviour and suggesting items purchased by similar users.

### 3. Truncated SVD
Truncated SVD is used as a model-based collaborative filtering method. It reduces the user-item interaction matrix into latent factors to capture hidden user preferences and item characteristics.

### 4. Co-Occurrence Recommendation
This method recommends items that are strongly associated with products already purchased by the user, based on transaction-level co-occurrence counts.

### 5. Hybrid Recommendation
The final hybrid model combines multiple recommendation sources into one ranked output. This helps balance relevance, diversity, and coverage.

## Key Findings

This project shows that grocery recommendation can be approached from multiple complementary directions:

- frequent patterns provide interpretable basket-level relationships
- collaborative filtering captures similarity between users
- SVD models latent preferences
- co-occurrence methods provide simple but effective related-item suggestions
- hybrid recommendation improves system coverage and combines different types of recommendation evidence

An important finding from the notebook output is that the hybrid model achieved very strong **coverage** and **catalog diversity**, although its **Precision@3** was lower than the best individual pattern-based recommendation model. This suggests that the hybrid design improved recommendation breadth and system robustness, but still has room for tuning if the goal is to maximise top-k precision. :contentReference[oaicite:5]{index=5}
